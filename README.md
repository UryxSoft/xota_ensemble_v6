## Overview

This system detects whether a piece of text was written by a human or generated by one of six major AI models. It uses a **three-model soft-voting ensemble** trained with a custom **Multi-Level Contrastive Loss (MCL)** that explicitly enforces:

- A strong Human вҶ” AI decision boundary (Оҙ=3.0 weight)
- Per-model AI clustering (GPT samples cluster together, Claude samples cluster together, etc.)
- Inter-model AI separation (GPT embeddings pushed away from Claude embeddings)

Every text is prefixed with a **Flesch-Kincaid Grade Level** score before being fed to the models, giving them an explicit readability signal. AI models tend to write at a narrower, more uniform reading level than humans.

### 8.6 Execution Order
```
Sequential (required for dependency resolution):
  1. StylometricPlugin     (no deps, CPU, ~10ms)
  2. ReasoningPlugin       (no deps, CPU, ~5ms)  
  3. HallucinationPlugin   (no deps, CPU+optional spaCy, ~15ms)
  4. WatermarkPlugin       (no deps, GPU/CPU, ~200ms first call, ~50ms cached)
  5. ForensicPlugin        (requires 1+2+3+4, ~50ms + matplotlib if visuals)

Parallelizable groups (1,2,3 can run concurrently; 4 independently):
  Group A: [Stylometric, Reasoning, Hallucination]  вҖ” pure CPU, ThreadPoolExecutor safe
  Group B: [Watermark]                              вҖ” GPU access, run after Group A completes
  Group C: [Forensic]                               вҖ” depends on A+B
```

### Loss Function

Each training step combines two losses:

```
Loss = MCL(CLS_embedding, labels) + CrossEntropy(logits, labels, class_weights)
```

**MCL** operates on the CLS token embedding of the last hidden layer:

| Level | Positives | Negatives | Weight |
|---|---|---|---|
| L1 вҖ” Human/AI boundary | Human вҶ” Human | Human вҶ” AI | Оҙ = 3.0 |
| L2 вҖ” Same model cluster | GPT вҶ” GPT, Claude вҶ” Claude | AI вҶ” different AI | Оұ = 1.0 |
| L4 вҖ” Inter-model separation | Different AI models | AI вҶ” Human | Оі = 1.0 |

Temperature П„ = 0.3 (increased from 0.1 in v5.3 for smoother gradients).

## Step 4 вҖ” Calibration & Production Hardening

This step runs **once after training completes**. It fits per-model temperature
scaling on the saved test logits and enables the full false-positive protection
layer for production use.

### 4.1 вҖ” Why calibration matters

Neural networks are systematically overconfident. A model that outputs
`P(GPT) = 0.91` is not necessarily 91% reliable вҖ” it may be correct only 70%
of the time at that threshold. Temperature scaling (Platt calibration) divides
the logits by a learned scalar `T` before softmax, pulling probabilities toward
their true empirical accuracy.

In academic integrity contexts, a **false positive** (human accused of using AI)
is significantly worse than a false negative. `CalibratedDetector` adds 5 layers
of protection on top of the raw ensemble.

### 4.2 вҖ” Run calibration (once, CPU or GPU)

Requires Phase 4 to have completed вҖ” the `.npy` logit files must exist in
`ensemble_dir`.
```python
from ensemble_v6_calibrated import calibrate_from_saved_logits

calibrate_from_saved_logits(
    "/content/drive/MyDrive/your_project/ensemble_v6/"
)
```

### 4.5 вҖ” False positive tolerance profiles

Choose the profile based on the consequences of a wrong accusation:

| Profile | Min P(AI) | Model agreement | Use case |
|---|---|---|---|
| `strict` | 0.75 | **3/3 models** | Academic integrity, legal review |
| `moderate` | 0.60 | 2/3 models | Content moderation, editorial review |
| `permissive` | 0.45 | 1/3 models | Research, bulk screening |

### 4.6 вҖ” 5-layer decision flow

Every call to `classify()` passes through these gates in order:
```
Ensemble probs (temp-scaled)
        вҶ“
1. Inconclusive band check
   If P(Human) вҲҲ [0.20, 0.55] вҶ’ INCONCLUSIVE

2. AI confidence threshold
   If P(AI) < 0.75 (strict) вҶ’ HUMAN

3. Agreement gating
   If fewer than 3/3 models vote AI (strict) вҶ’ INCONCLUSIVE

4. Margin check
   If top-1 prob вҲ’ top-2 prob < 0.25 вҶ’ INCONCLUSIVE

5. safe_for_action flag
   Only True when: strict + 3/3 agreement + P(AI) > 0.85
        вҶ“
   HUMAN | AI_DETECTED | INCONCLUSIVE
```

`safe_for_action=True` is the signal that the system is confident enough to
take consequential action (e.g. flagging a submission). `AI_DETECTED` with
`safe_for_action=False` means the system suspects AI but does not have
sufficient consensus for a formal accusation.

## Training Details

### Optimizer

`adamw_torch_fused` вҖ” the PyTorch fused implementation, ~15% faster than standard AdamW on CUDA. Parameters: `weight_decay=0.01`.

### Precision

`bf16=True` + `tf32=True` on all models. BFloat16 is numerically more stable than float16 for fine-tuning with contrastive losses.

### Dataloader

`num_workers=2` + `pin_memory=True` + `prefetch_factor=2` вҖ” overlaps data loading with GPU compute to prevent the GPU from idling between batches.

### Checkpointing strategy

`save_strategy="steps"` with `save_steps = max(10000, total_steps // 12)` вҖ” approximately 12 checkpoints per model training run. `save_total_limit=3` keeps only the 3 most recent on local SSD. All checkpoints are also copied to Drive by `StabilityCallback.on_save()`.

### No eval during training

`eval_strategy="no"` (nuclear mode) вҖ” eliminates ~15% of compute that would be spent on mid-training evaluation passes. Full evaluation runs after all models are trained in Phase 4.

## Troubleshooting

### "Los siguientes datasets tokenizados NO existen"

The training script found no tokenized data in Drive. Run preprocessing first:

```bash
# CPU runtime
!python ensemble_v6_preprocess.py
```

### Loss stuck at ~19вҖ“20 in early steps

Expected. The MCL contrastive term dominates early training and starts high with П„=0.3. The loss should begin dropping sharply after the warmup phase (~step 8,000 for DeBERTa). If still at 19+ past step 15,000, check that `CLASS_WEIGHTS` matches your actual dataset distribution.

### grad_norm spike > 500

Isolated spikes (1вҖ“2 occurrences) during training are normal for MCL with diverse batches. If grad_norm stays consistently above 400 for more than 5,000 steps, add gradient clipping:

```python
# In TrainingArguments:
max_grad_norm=1.0,
```

### torch_compile error on DeBERTa

DeBERTa's disentangled attention uses dynamic shapes that can cause compile failures. It is intentionally set to `torch_compile=False` in the config. If you see compile errors on ELECTRA or ModernBERT, set their `torch_compile` to `False` as well.

### W&B "Finishing previous run"

Harmless. W&B closes the run from the previous interrupted session before starting a new one. Training continues normally.

### Colab session killed before training completes

Re-run `ensemble_v6_train.py`. The script:
1. Skips models with a completed `config.json` in Drive
2. Resumes the in-progress model from its latest Drive checkpoint
3. Continues with remaining models

No data is lost.

### VRAM shows only 1вҖ“2 GB in logs

The `StabilityCallback` logs VRAM between steps, not during the forward pass. The actual peak VRAM (~22 GB) occurs inside the step and is not captured by the log. If you want to see the true peak, add `torch.cuda.max_memory_allocated()` to the callback.
